Não adicionar marcações de status (✅ Concluído, ❌ Pendente, etc.) nas descrições de user stories no ADO. A description é apenas para descrever o que deve ser feito.

**Why:** O usuário corrigiu isso — a seção de description deve ser puramente descritiva do trabalho necessário, sem tracking de progresso.

**How to apply:** Ao atualizar descriptions de work items, listar apenas as tarefas/requisitos sem indicar se já foram feitas ou não.
